# idDOS v1.0 — Ball заменён на Breakout game

## Что изменено

В LAB-приложении `ball` старый IMU-тест с прыгающим шариком заменён на полноценную мини-игру в стиле Breakout/Arkanoid, адаптированную под экран Cardputer-Adv 240x135.

## Управление

- Наклон влево/вправо: движение платформы.
- Наклон вперёд: платформа поднимается вверх примерно до 1/3 игровой зоны, выполняя активный отбивающий ход.
- ENTER или SPACE: запуск мяча / рестарт после проигрыша или победы.
- FN + ESC: выход назад в LAB.

## Игровое поле

- 4 линии кирпичей.
- 8 кирпичей в каждой линии.
- Всего 32 кирпича — оптимальный баланс для маленького экрана: поле выглядит заполненным, но не превращается в кашу.
- Вверху HUD: название BALL, очки, жизни, подсказка выхода.

## Техническая логика

- Приложение осталось неблокирующим: игра обновляется из `shellUpdate()` через `updateBallApp()`.
- Нет `while(true)`, нет захвата системы.
- Частота кадров ограничена примерно 60 FPS через `millis()`.
- Кирпичи хранятся в `bool bricks[4][8]`, без heap-аллоцирования.
- Физика простая и дешёвая: координаты `float`, отражения по стенам/платформе/кирпичам.
- Скорость мяча ограничена `BALL_SPEED_MAX`, чтобы игра не становилась неуправляемой.

## Файлы

- `src/shell/shell_ble_ball_update.cpp` — новая игра.
- `src/shell/shell_menus.cpp` — тег LAB-пункта изменён с `<IMU>` на `<GAME>`.

## Проверка на железе

Нужен реальный прогон на Cardputer-Adv:

1. Собрать проект через PlatformIO.
2. Залить на устройство.
3. Открыть `LAB -> ball`.
4. Проверить оси IMU:
   - наклон влево/вправо должен двигать платформу влево/вправо;
   - наклон вперёд должен поднимать платформу.
5. Если ось движения инвертирована — поменять знак в строке:
   `paddleX += -ax * 5.8f;`
6. Если forward-push инвертирован — поменять знак в строке:
   `float push = ay * PADDLE_PUSH_MAX;`
