# idDOS Ball / Breakout: green no-flicker patch

## Что изменено

Патч меняет только локальное приложение Ball/Breakout в файле:

```text
src/shell/shell_ble_ball_update.cpp
```

Ядро, boot, shell API, SD, audio, keyboard/IMU-драйверы не изменялись.

## Почему мерцало

Предыдущая версия каждый кадр делала полную перерисовку игровой области:

```cpp
d.fillRect(0, GAME_TOP, GAME_W, GAME_H - GAME_TOP, BLACK);
drawBricks();
```

На ST7789 240x135 это визуально заметно: каждый кадр экран сначала стирался, потом заново рисовались кирпичи, платформа и шар.

## Как исправлено

Теперь используется dirty-rectangle рендеринг:

- статическое поле и кирпичи рисуются один раз;
- HUD перерисовывается только при изменении очков/жизней;
- старые позиции мяча и платформы затираются маленькими прямоугольниками;
- новая позиция мяча и платформы рисуется поверх;
- уничтоженный кирпич затирается только один раз в момент попадания;
- центральное сообщение обновляется только при смене состояния.

## Стиль

Цветовая схема приведена к системному стилю idDOS:

- основной цвет: GREEN;
- фон: BLACK;
- акценты: WHITE и DARKGREY;
- кирпичи: оттенки зелёного.

## Управление

- наклон влево/вправо — движение платформы;
- наклон вперёд — активное движение платформы вверх;
- ENTER/SPACE — запуск или рестарт;
- FN+ESC — выход в LAB.
